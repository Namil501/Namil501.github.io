## 期間

2025-04 〜 2025-06

## 役割

メンバー（5名体制）

## 担当

- 設計書ベースの機能実装
- 単体テスト設計書作成とテスト実施

## 技術

- Java
- SpringBoot
- Oracle（PL/SQL）
- Windows

## 成果

- 設計書に基づく追加機能を期限内に実装
- 単体テスト仕様を作成し、実装品質の担保に寄与

## 学び

- 実装前に例外系ケースを洗い出すと、テスト仕様の抜け漏れを減らせる
- SpringBootは最小構成でもログ・例外設計を先に決めるべき

## 再現ポイント（運用/品質）

- 実装開始前に「入力バリエーションと異常系」をテンプレートで先出しする
- 単体テスト観点をPRテンプレートに含め、レビューで確認を必須化する
- DB変更を伴う場合、SQLレビューとアプリ影響確認をセットで運用する
